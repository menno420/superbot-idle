## 💡 Session idea

The corpus pins v1→v2 byte-exactly, but the FROZEN v1 field table lives
only as prose + `persistence._CURRENT_KEYS - {"milestones"}` derivation.
When v3 lands, the natural move is a `_FROZEN_FIELDS: dict[int,
frozenset]` table in `persistence.py` (v1, v2, ...) that the generator
reads instead of deriving — then a v3 PR that edits the v2 field set
in place (instead of bumping) reds THREE ways: the corpus, the field
table, and the migration registry. Guard recipe: add the table next to
`_CURRENT_KEYS`, point `tools/gen_save_vectors.py::V1_FIELDS/V2_FIELDS`
at it, pin `_FROZEN_FIELDS[STATE_VERSION] == _CURRENT_KEYS` in
`tests/test_persistence.py`.
